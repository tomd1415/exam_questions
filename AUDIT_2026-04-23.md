## Fixtures count — intentional shortfall

The chunk 3h plan text specifies 30 fixtures per prompt authored from anonymised
real pupil submissions. The repository ships 5 per prompt as a synthetic seed.
This is **deliberate** and documented in [prompts/eval/README.md](prompts/eval/README.md) —
the remaining 25 per prompt are a chunk 3i pilot deliverable, because real
pupil submissions do not exist yet. This is not a regression.

## Latent bug uncovered during the first live OpenAI call

While wiring up `npm run seed:ai-feedback-live` (see
[TEST_USERS.md](TEST_USERS.md) §Pupil AI-feedback demos) the first real call
against `mark_open_response v0.1.0` came back `HTTP 400 invalid_json_schema`:

> `'required' is required to be supplied and to be an array including every key in properties. Missing 'suggested_next_question_type'.`

Root cause: `FAMILY_B_OUTPUT_SCHEMA` in
[src/services/prompts_bootstrap.ts](src/services/prompts_bootstrap.ts) had
three optional properties (`notes`, `feedback_for_teacher.suggested_misconception_label`,
`feedback_for_teacher.suggested_next_question_type`) declared in `properties`
but omitted from `required`. OpenAI's Structured Outputs strict mode rejects
this — every property must appear in `required`, and optional fields are
expressed by giving them a nullable type (`['string', 'null']`) instead.

This bug had been invisible because every automated test uses a stub
`LlmClient`; no part of the existing CI path makes a real OpenAI call.

**Fix.** Added the three fields to `required` and changed their type to
`['string', 'null']`; marked the snapshot consideration in a comment so a
future edit doesn't regress it. Updated
[src/services/marking/llm.ts](src/services/marking/llm.ts) `LlmFeedbackForTeacher`
to reflect that the suggestion fields may legitimately be `null` at runtime.
Updated the already-seeded `prompt_versions` rows in the dev DB in place
(the bootstrap seeder is one-shot by design, so a code fix alone does not
propagate to existing rows). With the fix, the live seeder produced a clean
`awarded` outcome on the first retry: 2/2 marks, confidence 0.90, 1p cost,
no safety-gate flags.
